Clinical Data Pipeline for FDA Submission

A configurable pipeline that ingests clinical datasets from varied sources, validates and transforms them against user-defined rules, and exports clean, audit-ready output. The entire workflow runs from a Slack command.

Python GCP Cloud Run Slack API GCS Docker GxP / FDA
7d → <1hr Dataset preparation

3–5d → <4hr Issue investigation
20+ hrs/wk Engineering time saved

Overview

Before this pipeline, the scientific team spent up to a week manually validating and preparing data before each FDA submission, followed by additional QA cycles. The pipeline replaced that entirely by encoding validation logic in configuration files rather than bespoke scripts, making it reusable across new data sources with no additional engineering work.

We exposed the pipeline through a Slack bot instead of building a separate UI. Users post a formatted command with cloud storage pointers in a private channel. The bot triggers a Cloud Run job and threads all validation errors, warnings, and output locations back to the original message.
